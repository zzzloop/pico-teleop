# PICO 遥操作 Isaac Lab：主机/服务器部署手册

## 1. 最终架构

本项目只支持下面这一套部署：

```text
PICO 头显与控制器
  → USB / PICO Connect
Windows 主机 Unity OpenXR 输入程序
  → TCP 10000
GPU 服务器 ros_tcp_endpoint
  → 服务器本机 ROS 2
GPU 服务器 pico_teleop_bridge
  → UDP 127.0.0.1:9765
GPU 服务器 Isaac Lab 2.3.2 / BRX042501 / LeRobot

GPU 服务器 Isaac Sim
  → WebRTC TCP 49100 + UDP 47998
Windows 主机 Isaac Sim WebRTC Streaming Client
```

控制和画面是两条独立链路。WebRTC 断开不会改变 PICO/ROS 控制协议；Unity TCP 断开会触发仿真看门狗并保持机器人。PICO 不直接访问服务器，Windows 不运行 ROS 2，ROS 2 DDS 不跨主机。

## 2. 固定目录和网络变量

本文统一使用：

| 名称 | 值 |
|---|---|
| Windows 项目 | `C:\Users\zrobot\Desktop\PICO遥操作` |
| Windows 参考项目 | `D:\IsaacLabCode` |
| Windows 主机 IP | `192.168.50.61` |
| GPU 服务器 IP | `192.168.50.227` |
| 服务器项目 | `/srv/pico_teleop` |
| 服务器 Isaac Lab | `/srv/IsaacLabCode` |
| 服务器数据 | `/srv/datasets` |
| Unity/ROS-TCP | TCP `10000` |
| Isaac WebRTC 信令 | TCP `49100` |
| Isaac WebRTC 视频 | UDP `47998` |
| bridge/Isaac | UDP `127.0.0.1:9765` |

Windows 主机和服务器已经位于 `192.168.50.0/24` 局域网。Unity 和 WebRTC 均固定连接 `192.168.50.227`；文中只有服务器登录用户名 `SERVER_USER` 需要按实际账户替换。

## 3. 服务器一次性准备

### 3.1 系统和 GPU

服务器使用与 Isaac Lab 2.3.2/Isaac Sim 5.1 匹配的 Linux 环境，并安装：

- NVIDIA 驱动和支持 NVENC 的 NVIDIA GPU；A100 没有 NVENC，不能用于该 WebRTC 显示链路。
- Isaac Sim 5.1 对应运行时和 Isaac Lab 2.3.2。
- ROS 2 Humble desktop/base、`colcon`、`rosdep`。
- `ffmpeg`，用于 LeRobot 视频写入。

NVIDIA 官方说明中，WebRTC 每个 Isaac Sim 实例同一时间只允许一个客户端，并建议客户端与服务器位于同一网络。

### 3.2 创建目录

在服务器执行：

```bash
sudo mkdir -p /srv/pico_teleop /srv/IsaacLabCode /srv/datasets
sudo chown -R "$USER:$USER" /srv/pico_teleop /srv/IsaacLabCode /srv/datasets
```

### 3.3 部署 Isaac Lab 2.3.2

在服务器以 Linux 方式安装 Isaac Lab，不要把 Windows 的 `_isaac_sim` 链接、conda 环境或二进制目录直接复制到服务器。代码目录固定为 `/srv/IsaacLabCode`。安装完成后验证：

```bash
cd /srv/IsaacLabCode
./isaaclab.sh -p -c "import isaaclab; print('Isaac Lab import OK')"
```

将 Windows `D:\IsaacLabCode` 中本项目依赖的自定义内容同步到服务器同名位置：

```text
D:\IsaacLabCode\BRX042501      → /srv/IsaacLabCode/BRX042501
D:\IsaacLabCode\teleop\assets → /srv/IsaacLabCode/teleop/assets
```

### 3.4 同步本项目

在 Windows PowerShell 执行：

```powershell
scp -r "C:\Users\zrobot\Desktop\PICO遥操作\." SERVER_USER@192.168.50.227:/srv/pico_teleop/
```

每次本地主机代码更新后重新同步，再在服务器重新构建 ROS 工作区。

同步后运行服务器预检：

```bash
cd /srv/IsaacLabCode
./isaaclab.sh -p /srv/pico_teleop/scripts/check_environment.py \
  --isaaclab-root /srv/IsaacLabCode
```

结果中应为 Isaac Lab `2.3.2`、`urdf_ok: true`，并显示 LeRobot 包版本。

### 3.5 构建服务器 ROS 2 工作区

```bash
cd /srv/pico_teleop/ros2_ws
source /opt/ros/humble/setup.bash
sudo rosdep init 2>/dev/null || true
rosdep update
rosdep install --from-paths src --ignore-src -r -y
colcon build --symlink-install
source install/setup.bash
ros2 pkg executables pico_teleop_bridge
ros2 pkg executables ros_tcp_endpoint
```

项目内已包含 Unity 官方 `ROS-TCP-Endpoint` 的 `main-ros2` 分支，不需要再 clone 第二份。

### 3.6 安装 LeRobot

LeRobot 必须安装在实际运行 `./isaaclab.sh -p` 的 Python 中。v3 使用 LeRobot `0.4.x`：

```bash
cd /srv/IsaacLabCode
./isaaclab.sh -p -m pip install "lerobot>=0.4,<0.5"
./isaaclab.sh -p -c "import lerobot; print(lerobot.__version__)"
```

要原生写 v2.1 时，使用单独的 Isaac Lab Python 环境安装 LeRobot `0.3.x`，并运行同一份项目代码：

```bash
./isaaclab.sh -p -m pip install "lerobot>=0.3,<0.4"
```

不要在同一个 Python 环境同时安装两代 LeRobot，也不要把 v2.1 和 v3 写入同一个 `record_root`。项目的 `--lerobot_format` 会检查实际 dataset format，不匹配时拒绝开始录制。

## 4. 网络和防火墙

服务器只允许 Windows 主机私网地址访问三项运行端口：

```bash
sudo ufw allow from 192.168.50.61 to any port 10000 proto tcp
sudo ufw allow from 192.168.50.61 to any port 49100 proto tcp
sudo ufw allow from 192.168.50.61 to any port 47998 proto udp
```

不要开放 UDP 9765。启动后用以下命令确认：

```bash
ss -lntp | grep -E ':10000|:49100'
ss -lunp | grep -E ':47998|:9765'
```

预期：endpoint 监听 `0.0.0.0:10000`；Isaac WebRTC 使用 49100/47998；仿真 UDP 只显示 `127.0.0.1:9765`。

## 5. Windows 主机一次性准备

1. 安装 PICO Connect，用 USB 把 PICO 连接为 Windows PCVR/OpenXR 设备。
2. 创建 Windows x86_64 Unity OpenXR 输入程序，加入 `PicoRosPublisher.cs`。
3. Unity `Robotics → ROS Settings` 设置 `ROS2`、`192.168.50.227`、端口 `10000`。
4. 安装与 Isaac Sim 5.1 对应的 Isaac Sim WebRTC Streaming Client。

完整 Unity 步骤见 [PICO Unity 接入](PICO_Unity接入.md)。PICO 头显只维持 tracking，操作员在 Windows 显示器上看 WebRTC 窗口并使用两个控制器。

## 6. 每次启动：严格按顺序

### 6.1 服务器终端 1：ROS-TCP 和 bridge

```bash
source /opt/ros/humble/setup.bash
source /srv/pico_teleop/ros2_ws/install/setup.bash
ros2 launch pico_teleop_bridge pico_teleop.launch.py
```

日志必须出现类似：

```text
Starting server on 0.0.0.0:10000
PICO bridge ready: ROS 2 -> udp://127.0.0.1:9765 at 60.0 Hz
```

### 6.2 服务器终端 2：Isaac Lab、WebRTC、v3 记录

```bash
cd /srv/IsaacLabCode
./isaaclab.sh -p /srv/pico_teleop/scripts/run_sim.py \
  --livestream 2 \
  --enable_cameras \
  --device cuda:0 \
  --udp_port 9765 \
  --urdf_path /srv/IsaacLabCode/BRX042501/BRX042501_wheel_4cams.urdf \
  --teleop_assets_root /srv/IsaacLabCode/teleop/assets \
  --camera_width 640 --camera_height 360 \
  --camera_hz 15 --record_fps 15 \
  --record_root /srv/datasets/brx_pico_v3 \
  --record_repo_id local/brx042501_pico_v3 \
  --lerobot_format v3 \
  --record_task "Pick up the target block and place it in the bucket."
```

`--livestream 2` 是 Isaac Lab 2.3.x 的私网 WebRTC 模式，并会隐式启用 headless。`--enable_cameras` 仍必须保留，因为本项目还要渲染四路数据相机。

无录制调试时删除 `--record_root`、`--record_repo_id`、`--lerobot_format` 和 `--record_task` 四项；控制与 WebRTC 不受影响。

等待日志出现仿真 ready、`[PICO][UDP] listening on udp://127.0.0.1:9765`，并让首次 shader/cache 初始化完成。

### 6.3 Windows：启动 PICO 输入

1. 启动 PICO Connect并确认头显、左右手柄已追踪。
2. 启动构建好的 `PicoTeleopInput.exe`。
3. 在服务器检查：

```bash
source /opt/ros/humble/setup.bash
source /srv/pico_teleop/ros2_ws/install/setup.bash
ros2 topic hz /pico/left_controller/pose
ros2 topic hz /pico/right_controller/pose
```

### 6.4 Windows：连接 WebRTC

1. 启动 Isaac Sim WebRTC Streaming Client。
2. Server 填 `192.168.50.227`。
3. 点击 Connect，等待 Isaac Sim 界面出现。

同一 Isaac Sim 实例不要打开第二个 WebRTC 客户端。WebRTC 窗口负责画面；Unity 输入程序必须继续在后台运行。

## 7. 一次标准采集流程

1. WebRTC 画面中确认机器人、桌面、桶、方块和四相机均已初始化。
2. 双手控制器放到舒适的中立位置。
3. 按 A：进入 running，建立双手 anchor；有 `record_root` 时自动开始 episode。
4. 移动控制器遥操作；左右 trigger 控制夹爪。
5. 按 B：pause。机器人保持、WebRTC 和 ROS 状态继续，但不写 LeRobot 帧。
6. 把控制器移动到新的舒适位置后按 A：resume；下一有效 pose 自动重新标定，不产生跳变。
7. 完成任务后按 Y：stop，保持机器人并保存 episode。
8. 下一条 episode 再按 A。需要换任务文本时，在开始前执行：

```bash
ros2 topic pub --once /pico_teleop/task std_msgs/msg/String \
  "{data: 'Pick up the blue block and place it in the bucket.'}"
```

默认 A 已经自动控制 episode 生命周期。X 是手动 record toggle，只有使用 `--no_auto_record_on_start` 时才需要。

## 8. ROS 服务控制

PICO 按键之外，可在服务器验证状态机：

```bash
ros2 service call /pico_teleop/start std_srvs/srv/Trigger '{}'
ros2 service call /pico_teleop/pause std_srvs/srv/Trigger '{}'
ros2 service call /pico_teleop/resume std_srvs/srv/Trigger '{}'
ros2 service call /pico_teleop/stop std_srvs/srv/Trigger '{}'
ros2 topic echo --once /pico_teleop/status
```

`pause` 不是停止 Isaac 仿真进程；它只冻结遥操作目标和录制 gate，因此 WebRTC 不会黑屏，服务也仍可响应。

## 9. 写 v2.1 数据

切换到安装 LeRobot `0.3.x` 的服务器 Isaac Lab Python 环境，使用独立目录：

```bash
cd /srv/IsaacLabCode
./isaaclab.sh -p /srv/pico_teleop/scripts/run_sim.py \
  --livestream 2 --enable_cameras --device cuda:0 \
  --urdf_path /srv/IsaacLabCode/BRX042501/BRX042501_wheel_4cams.urdf \
  --teleop_assets_root /srv/IsaacLabCode/teleop/assets \
  --record_root /srv/datasets/brx_pico_v2 \
  --record_repo_id local/brx042501_pico_v2 \
  --lerobot_format v2
```

v2.1 与 v3 的 feature key、四相机、23 维 state/action 契约完全一致，只有 LeRobot 官方 writer 产生的磁盘布局不同。详细 schema 见 [LeRobot 数据格式](LeRobot数据格式.md)。

## 10. 数据校验

每次采集结束，在对应 LeRobot 环境运行：

```bash
cd /srv/pico_teleop
/srv/IsaacLabCode/isaaclab.sh -p -m pico_isaaclab.validate_dataset \
  /srv/datasets/brx_pico_v3 --repo_id local/brx042501_pico_v3
```

必须确认：

- 四个 image/video feature 都存在且为 `[360,640,3]`。
- `observation.state` 和 `action` 均为 float32 `[23]`。
- `dropped_frames == 0`，`error == null`。
- episode 帧数大于 0，官方 `LeRobotDataset` loader 能打开首尾帧。

## 11. 正确关闭

1. 若 episode 仍 active，先按 Y 或调用 `/pico_teleop/stop`。
2. 需要正式封存数据时调用 `/pico_teleop/record_finalize`。
3. 在 Isaac Lab 服务器终端按 `Ctrl+C`，等待 recorder finalize/close 完成。
4. 关闭 Windows WebRTC Client。
5. 关闭 Unity 输入程序和 PICO Connect。
6. 最后停止 ROS launch。

不要直接 `kill -9` Isaac Lab；v3 的 Parquet/video writer 在 `finalize()` 前可能尚未写完 footer。

## 12. 验收清单

### 通信

- `ss -lntp` 显示 TCP 10000。
- Unity 连接后服务器能看到四个 `/pico/*` 话题。
- `/pico_teleop/status` 中 `transport.connected=true`。
- UDP 9765 只绑定 `127.0.0.1`。

### WebRTC

- 主机 `192.168.50.61` 连接 `192.168.50.227` 后显示 Isaac Sim。
- 只有一个客户端。
- 服务器 GPU 支持 NVENC，TCP 49100/UDP 47998 对主机开放。

### 遥操作安全

- A 后移动平滑，没有首次跳变。
- B 后机器人保持且 `recording_gate=false`。
- 恢复后重新标定，没有大步跳变。
- 拔掉 PICO/停止 Unity 后约 350 ms 内进入 safety hold。

### 数据

- Y 后 episode 保存成功。
- 四路图像时间同步。
- state/action 均为 23 维。
- v2 和 v3 使用不同 root，且格式校验通过。

## 13. 故障排查

- `Unity connection refused`：先确认 ROS launch 仍在运行，再检查服务器 `0.0.0.0:10000` 和防火墙。
- ROS 有 topic、机器人不动：确认 Isaac 日志绑定 `127.0.0.1:9765`，检查 `/pico_teleop/status` 的 `session_state`、`hold_reason`、`packet_age_ms`。
- `tracking_lost`：恢复 PICO Connect/控制器 tracking；有效 pose 回来后按 A 或调用 calibrate。
- `command_timeout`：Unity 输入程序停止发布或 TCP 链路中断；不要放宽 timeout 掩盖网络故障。
- WebRTC 连接失败：确认只开一个客户端、服务器 GPU 有 NVENC、TCP 49100 和 UDP 47998 可达。
- WebRTC 正常但 PICO 不动：WebRTC 与控制链独立；检查 Unity ROS-TCP 10000，不要排查 WebRTC 端口。
- PICO topic 正常但 WebRTC 黑屏：检查 Isaac `--livestream 2`、NVENC、WebRTC 端口和 shader 初始化日志，不要修改 ROS。
- `format v2 requested, but ... v3`：当前 LeRobot 环境与 `--lerobot_format` 不一致，切换对应环境和独立数据目录。
- `Refusing to save an empty episode`：开始后尚未完成标定或四相机没有产出有效同步帧。
- `dropped_frames > 0`：降低相机分辨率/FPS，确认服务器磁盘写入速度，再重采该 episode。
